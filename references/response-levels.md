# Response Levels System (分级响应体系)

基于拿破仑"大军团"原则：速度就是一切，但速度需要分级。

---

## 核心理念

> "我可以输掉一场战役，但绝不能输掉一分钟。"
>
> 不同信号强度需要不同响应速度和资源投入。过度反应浪费资源，反应不足错失机会。

---

## 响应等级定义

### 🔴 Level 5: 全面战争 (Total War)

**触发条件**：
- Battle Index ≥ 8
- 五事前兆 5/5 全亮
- 竞争窗口 < 72小时

**资源投入**：10人日

**响应时间**：24小时内完成所有动作

**标准动作清单**：
```
□ T+0h:   触发确认，启动作战室
□ T+1h:   完成 Flash Report
□ T+2h:   域名检查+注册（如可用）
□ T+4h:   落地页骨架上线
□ T+6h:   社交媒体占位帖发布
□ T+8h:   完成 Full Report
□ T+12h:  第一篇核心内容发布
□ T+24h:  长尾词内容矩阵完成（5篇）
□ T+48h:  视频内容发布
□ T+72h:  复盘+持续优化
```

**审批要求**：需项目负责人确认

**Sprunki 假设**：如果在 2024年9月 以 Level 5 响应
- 预期结果：可能占据前3名自然搜索位置
- 错过代价：现在进入为时已晚

---

### 🟠 Level 4: 有限干预 (Limited Intervention)

**触发条件**：
- Battle Index 6-8
- 五事前兆 4/5 亮起
- 竞争窗口 3-7天

**资源投入**：5人日

**响应时间**：48小时内完成核心动作

**标准动作清单**：
```
□ T+0h:   触发确认
□ T+2h:   完成 Flash Report
□ T+4h:   域名检查（不急于注册）
□ T+8h:   社交媒体占位帖发布
□ T+24h:  完成 Full Report
□ T+48h:  第一篇核心内容发布
□ T+72h:  评估是否升级到 Level 5
```

**审批要求**：自主决策，事后报备

---

### 🟡 Level 3: 代理战争 (Proxy War)

**触发条件**：
- Battle Index 5-6
- 五事前兆 3/5 亮起
- 竞争窗口 1-2周

**资源投入**：2人日

**响应时间**：7天内完成

**标准动作清单**：
```
□ T+0h:   触发确认
□ T+4h:   完成 Flash Report
□ T+24h:  决定是否继续
□ T+48h:  使用模板快速产出1篇内容
□ T+72h:  发布社交媒体测试帖
□ T+7d:   评估效果，决定升级/降级/退出
```

**审批要求**：无需审批

**关键原则**：用最小投入测试市场反应

---

### 🔵 Level 2: 外交施压 (Diplomatic Pressure)

**触发条件**：
- Battle Index 4-5
- 五事前兆 2/5 亮起
- 不确定性高

**资源投入**：0.5人日

**响应时间**：持续监控

**标准动作清单**：
```
□ T+0h:   记录信号
□ T+1h:   发布1条社交媒体占位帖
□ 每日:   监控 Trends 变化
□ 每周:   评估是否升级
```

**审批要求**：无

**目的**：占坑不投入，等待信号明确

---

### ⚪ Level 1: 战略观望 (Strategic Observation)

**触发条件**：
- Battle Index < 4
- 五事前兆 ≤ 1/5 亮起
- 或已过最佳窗口（如 Sprunki 现状）

**资源投入**：0

**响应时间**：无

**标准动作清单**：
```
□ 记录到案例库
□ 提炼经验教训
□ 设置复苏监控（如适用）
```

**审批要求**：无

**Sprunki 现状**：应归为 Level 1，仅作案例研究

## 升级与降级规则

### 升级条件

| 当前等级 | 升级触发 | 升级到 |
|----------|----------|--------|
| L1 | Trends 周增 >300% | L2 |
| L2 | 五事新增1个亮起 | L3 |
| L3 | 测试内容表现超预期 (CTR >5%) | L4 |
| L4 | 竞争窗口收窄至 <72h | L5 |

### 降级条件

| 当前等级 | 降级触发 | 降级到 |
|----------|----------|--------|
| L5 | 72h后竞争饱和 | L3 (维护) |
| L4 | Trends 连续3天下降 | L2 |
| L3 | 测试内容表现差 (CTR <1%) | L1 |
| L2 | 2周无信号变化 | L1 |

## 作战室启动协议 (Level 5 专用)

当触发 Level 5 时：

```
作战室启动清单：

1. 人员召集
   □ 确认可用人员
   □ 分配角色（内容/技术/监控）
   □ 建立沟通频道

2. 情报汇总
   □ 整合所有已知信息
   □ 识别关键缺口
   □ 分配情报采集任务

3. 目标锁定
   □ 确定核心关键词（1-3个）
   □ 确定长尾词矩阵（10-20个）
   □ 确定内容优先级

4. 执行启动
   □ 按清单执行
   □ 每2小时同步进度
   □ 实时调整策略

5. 战后复盘
   □ T+72h 完成复盘
   □ 记录经验教训
   □ 更新案例库
```
